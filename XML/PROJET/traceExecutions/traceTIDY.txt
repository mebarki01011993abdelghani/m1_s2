evaluer et determiner le degre de confiance que l'on peut attribuer

du cours, chaque niveau est passe en revue, de facon generale dans

verifier. L'accent est surtout mis sur les niveaux les plus eleves,

formelle. Dans le cours, l'objectif principal est de montrer qu'un
programme est fiable par rapport aux specifications initiales. Pour
cela, le cours presente : le systeme de specification formelle Z
(notation mathematique basee sur la theorie des ensembles et sur la
logique du premier ordre), l'atelier de genie logiciel B (dont la
notation Z est a l'origine ; les supports de Jean-Raymond Abrial
sont utilises dans un cours), les differents systemes de types qui
existent et en particulier, un systeme de type evolue au travers du
systeme Coq (base sur la theorie du Calcul des Constructions qui
est un lambda-calcul type d'ordre superieur).</li>

<div class="contenu">

plateforme JEE, le couplage entre EJB et base de donnees
relationnelle pour la manipulation des donnees metier ( Java
Persistance API ), l'architecture MVC des applications Web avec
Spring MVC et Struts, les principes de securisation des
applications Web les technologies JSF (Java Server Faces) et GWT
(Google Web Toolkit) pour la realisation d'applications Web riches,
les techniques de conception et de realisation des Web Services
(SOAP, XML-RPC, WSDL).</li>
</ul>

<div class="titre">Securite des systemes embarques</div>
<div class="contenu">
<ul>
<li>Identifiant : ENSINCU35</li>
<li>Nombre de credits 3</li>
<li>Resume : But de ce module est de familiariser les etudiants
avec les principes de la norme USB et de leur donner les moyens
pour developper des pilotes USB côte hôte et côte peripherique. La
norme USB etant vaste, on se concentrera sur deux classes
particulieres, les peripheriques HID (human interface devise) et
les peripheriques CCID (circuit card interface devices,
c'est-a-dire des lecteurs de cartes a puce sans et avec contact
pour des applications de securite). Contenu du cours : Introduction
a USB : architecture materiel et logiciel, classes de
peripheriques, classes de transferts, nouveautes dans USB 3.0
Protocole de communication USB, detaille, schema de transactions,
terminaisons, types de transfert ; descripteurs et enumerations
Details de la classe classe HID : exemples simples (souris,
clavier) et exemples plus complexes et applications USB pratique
cote peripherique : ce qu'il faut faire, comment utiliser les
bibliotheques USB, ce qu'il faut ajouter, interruptions et threads.
USB pratique cote hôte (Linux) : ce qu'il faut faire, comment
utiliser libusb, comment ecrire des pilotes comme extensions de
kernel, ... Introduction aux smart cards et a la classe CCID.</li>
</ul>
</div>

de contrôle), l'architecture des applications 3-tiers, Couche de
presentation et interfaces Web : les technologies de mise en œuvre
des applications WEB (Servlet, JSP, Tomcat, Glassfish).</li>

des outils de prototypage, de test et de suivi de logiciels, avec
des methodologies normalisees. Elle apporte des competences fortes
en securite informatique dans les domaines lies a l'echange
d'information (Web-services, protocoles de communication et
cryptographiques,..) et a la communication entre systemes (reseaux
heterogenes et applications embarquees ou distribuees).</div>

methodes de securite informatique dans un environnement distribue
et ouvert Savoir specifier, concevoir et realiser un systeme
logiciel et materiel complexe Programmation de systemes
embarques</div>
<div xmlns="" class="titre">Connaissances a acquerir</div>
<div xmlns="" class="contenu">L'objectif de la specialite FSIL est

valider et mettre en exploitation des logiciels. Pour ce faire, ils
devront connaitre les concepts, les outils et les methodes lies au
developpement de logiciels. Les aspects de qualite, de securite et
de bonne integration dans des environnements informatiques
complexes seront les points forts de la formation.</div>
